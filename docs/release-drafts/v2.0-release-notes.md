# darklab_shell v2.0 Release Notes

For the full list of changes included in this release, see the v2.0 section in [CHANGELOG.md](../../CHANGELOG.md).

## Highlights

- Interactive PTY mode adds a real terminal surface for screen-oriented tools such as `mtr --interactive`, `ffuf --interactive`, and `masscan --interactive`, backed by registry-owned settings and Redis-friendly control routing.
- When PTY mode is enabled, the browser preloads the terminal rendering assets during startup so the first interactive command opens with less visible delay.
- PTY terminal assets recover from failed loads on retry, and open PTY terminals now pick up theme changes without requiring a reload.
- Mobile terminal mode now returns a clear desktop-only message for `--interactive` commands instead of opening a PTY surface that cannot reliably accept mobile keyboard input.
- The PTY modal is scoped to the tab that owns the interactive run, so other tabs remain usable while the terminal keeps running.
- PTY modals now have a Hide control that leaves the process running, shows a reopen indicator in the owning tab, and Status Monitor gives clear guidance when a PTY is still active but cannot be reattached after reload yet.
- If a second PTY start is rejected because one is already active, the original live terminal stays attached to its tab.
- `Ctrl+C` in the PTY modal now behaves like a normal terminal interrupt, while the modal kill and close buttons still use the standard confirmation flow.
- PTY input and resize requests now use the shared rate limiter, and large pasted/input payloads are trimmed by UTF-8 byte length before being sent.
- Interactive PTY commands now receive a narrow, vetted environment instead of a tiny hardcoded environment or the full server process environment.
- If the required server-side terminal capture dependency is unavailable, PTY startup now reports the dependency issue clearly instead of silently degrading saved output quality.
- PTY sessions preserve live terminal formatting in the modal while the parent tab receives the saved static transcript and exit status when the run finishes.
- SQLite run previews are now byte-bounded with `output_preview_max_mb`, preventing huge single-line scanner output from bloating the history database while preserving larger output in full-output artifacts when enabled.
- Startup history queries now use targeted SQLite indexes so large history databases do not delay the Recent and Workflows rail menus.
- History restore now distinguishes runs by run identity, so restoring a command no longer jumps to an unrelated existing tab with the same command text.
- The command composer now blocks macOS double-space period substitution from changing terminal input.
