# darklab_shell v2.0 Release Notes

For the full list of changes included in this release, see the v2.0 section in [CHANGELOG.md](../../CHANGELOG.md).

## Highlights

- Project workspace groundwork adds the database relationship model, active project context, automatic saved-run and snapshot linking, project-aware history filters, and first backend routes for future case-style grouping across projects, runs, snapshots, workspace files, artifacts, targets, findings, labels, annotations, and packages.
- Interactive PTY mode adds a real terminal surface for screen-oriented tools such as `mtr --interactive`, `ffuf --interactive`, and `masscan --interactive`, backed by registry-owned settings, dedicated PTY routes, Redis-friendly control routing, and a single-worker local fallback.
- The live PTY surface uses a tab-scoped xterm.js modal with fit-addon support, startup asset preloading, asset-load retry handling, theme refresh, reusable tab overlays, hidden-tab resize suppression, desktop-only mobile guardrails, and the same close/kill confirmation flow used by normal running tabs.
- PTY runs support bounded concurrency per browser session, defaulting to four active PTYs, while keeping each live terminal scoped to the tab that started it.
- Reload recovery and Status Monitor Attach restore active PTY runs from bounded ANSI terminal snapshots, preserving colors, text attributes, cursor position, and the latest visible screen when server-side terminal capture is available. Redis-backed deployments publish distributed, rate-capped snapshots so any worker can serve reattach without sticky routing.
- Completed PTY runs preserve live terminal formatting in the modal while appending a saved static transcript and exit status to the parent tab. The command registry controls transcript shaping, so `mtr` saves the final visible frame while `ffuf` and `masscan` save scrolled findings without common transient progress/status redraw lines.
- Interactive PTY mode ships with first-pass safety guardrails: explicit `input_safety` profiles, native `Ctrl+C` forwarding, rate-limited input and resize requests, UTF-8 byte-aware input trimming, workspace-CWD-aware validation before spawn, a vetted command environment, clear server-side terminal capture dependency errors, stale state cleanup, and actionable snapshot failure statuses.
- SQLite run previews are now byte-bounded with `output_preview_max_mb`, preventing huge single-line scanner output from bloating the history database while preserving larger output in full-output artifacts when enabled.
- Startup history queries now use targeted SQLite indexes so large history databases do not delay the Recent and Workflows rail menus.
- History restore now distinguishes runs by run identity, so restoring a command no longer jumps to an unrelated existing tab with the same command text.
- The command composer now blocks macOS double-space period substitution from changing terminal input.
- Recent value capture now preserves complete IPv4 addresses entered in domain-style argument slots.
