# v1.7 Release Merge Request

## Summary

v1.7 focuses on making session Files feel more complete from both the modal and the terminal. Users can now move or rename workspace files and folders, drag files into folders, and use simple `*` patterns for everyday file list/move/delete flows without opening raw shell filesystem access.

For the full list of changes included in this release, see the v1.7 section in [CHANGELOG.md](../../CHANGELOG.md).

## Added

- Session Files move/rename support in the Files modal:
  - row-level **move** action
  - drag-and-drop from file rows onto folder rows
  - app-native prompt and confirmation copy
  - refreshed workspace payload after each move
- Terminal-native move commands:
  - `file move <source> <destination>`
  - `mv <source> <destination>` alias
  - root-aware relative paths from the tab's current workspace folder
- Backend move route:
  - `POST /workspace/files/move`
  - shared `move_workspace_path()` validation and movement logic
  - structured `WORKSPACE_FILE_MOVE` log event
- Simple workspace glob support:
  - `file ls darklab-*`
  - `mv darklab-* darklab/`
  - `file delete scan-*`
  - one-segment matching, so patterns do not cross `/`
- Workspace autocomplete for move flows:
  - live source suggestions from session files and folders
  - destination folder suggestions, including `/`
  - destination filtering so the source and its children are not suggested as targets

## Changed

- Files autocomplete now lets `app/builtin_autocomplete.yaml` own the visible `file list <folder>` and `file ls <folder>` grammar entries, while runtime autocomplete only adds live workspace values.
- Workspace built-ins now have matching browser and backend behavior for move and glob flows, which helps stale-browser and server-rendered command paths behave consistently.
- Test documentation now reflects the current suite totals: 2,400 total tests across pytest, Vitest, and Playwright.

## Fixed

- Removed duplicate autocomplete rows for `file list` and `file ls`.
- Multi-item move/delete flows now fail with clear messages when there are no matches, when a multi-move destination is not an existing folder, or when a folder delete needs `-r` / `-rf`.

## Validation

- Added pytest coverage for workspace move routes, invalid move paths, recursive folder move rejection, and glob matching.
- Added Vitest coverage for Files modal move actions, drag/drop, `file move`, `mv`, glob listing, glob moving, and confirmed glob deletion.
- Extended the Playwright Files workflow to move a file from the modal and move it back from the terminal.
