# v1.7 Release Merge Request

## Summary

v1.7 improves the day-to-day shell workflow around session files and supported commands. Files can be moved from the modal or terminal, path autocomplete understands folders, supported commands have a real details surface, and several active-run edge cases are cleaned up.

For the full list of changes included in this release, see the v1.7 section in [CHANGELOG.md](../../CHANGELOG.md).

## Added

- Session Files move/rename support:
  - row-level **move** action in the Files modal
  - drag-and-drop from file rows onto folder rows
  - `POST /workspace/files/move`
  - `file move <source> <destination>`
  - `mv <source> <destination>` alias
- Simple workspace pattern support:
  - `file ls darklab-*`
  - `mv darklab-* darklab/`
  - confirmed `file delete scan-*`
  - one-segment matching, so patterns do not cross `/`
- Directory-aware workspace autocomplete:
  - suggestions inside typed folder prefixes such as `darklab/`, `../`, and `/darklab/`
  - trailing `/` on folder suggestions and inserts
  - immediate refresh after accepting a folder
  - source/destination-aware suggestions for `file move` and `mv`
- CWD-aware external workspace file flags:
  - `/runs` carries the active tab `workspace_cwd`
  - declared read/write workspace flag values resolve against that folder
  - parent paths can move upward inside the workspace but cannot escape it
  - packaged absolute paths such as `/usr/share/wordlists/...` are preserved
  - external flag autocomplete inserts CWD-relative file names instead of root-relative workspace paths
- Supported-command catalog:
  - one-sentence `description` metadata for every top-level external command in `commands.yaml`
  - `commands info <root> [subcommand]`
  - `/commands/catalog`
  - `/commands/catalog/<root>`
  - desktop/mobile Command Registry browser with search, category filters, and command rows
  - reusable command-details modal with examples, flags, and subcommands from the registry

## Changed

- Autocomplete placeholder guidance now uses explicit `hint_only: true` metadata and is no longer treated as a selectable menu item.
- The FAQ now points users to the Command Registry instead of rendering the full supported-command list inside a FAQ answer.
- Status Monitor uptime now ticks locally between `/status` polls instead of waiting for the next status response.
- Closing a running tab with **Keep running** now keeps that run detached after reload until the user manually attaches from Status Monitor.
- Files autocomplete now lets `app/builtin_autocomplete.yaml` own the visible `file list <folder>` and `file ls <folder>` grammar entries, while runtime autocomplete only adds live workspace values.
- Test documentation now reflects the current suite totals: 2,430 total tests across pytest, Vitest, and Playwright.

## Fixed

- Hidden prompt input and autocomplete are blocked while the active tab is running, preventing stray autocomplete menus from appearing away from the prompt.
- Command-created workspace config folders are repaired before list/read operations, so tool-managed files remain visible and readable through Files and `cat`.
- Oversized workspace files now show a toast before opening the viewer/editor modal and no longer leave Files stuck in loading state.
- Removed duplicate autocomplete rows for `file list` and `file ls`.
- Multi-item move/delete flows now fail with clear messages when there are no matches, when a multi-move destination is not an existing folder, or when a folder delete needs `-r` / `-rf`.

## Validation

- Added pytest coverage for workspace move routes, invalid move paths, recursive folder move rejection, glob matching, command catalog helpers/routes, workspace permission repair, and oversized-file handling.
- Added Vitest coverage for Files modal move actions, drag/drop, `file move`, `mv`, glob listing, glob moving, confirmed glob deletion, directory-aware autocomplete, hint-only autocomplete behavior, command registry browsing, active-run input blocking, detached-run reload behavior, and Status Monitor uptime ticking.
- Extended Playwright coverage for the Files workflow moving a file from the modal and moving it back from the terminal, plus FAQ-to-Command-Registry discovery.
