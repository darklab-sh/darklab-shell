# v1.6 Merge Request

## Summary

- What changed
  - added app-mediated per-session workspace storage, the Files modal, folder navigation, file view/edit/download/delete flows, dedicated new/edit/view modals, richer JSON/JSONL/CSV/TSV/XML/HTTP/text file previews with terminal-style search outside the scrollable preview, manual/auto-refresh for open previews, recursive folder delete with count-aware warnings, and terminal-native `file` / `cd` / `pwd` / `ls` / `cat` / `mkdir` / confirmed `rm` helpers with per-tab cwd tracking, terminal-like `ls | grep` pipe semantics, and compact mobile cwd placeholders
  - made workspace-aware scanner examples first-class: declared file input/output flags are rewritten into the guarded session workspace, workspace-only examples stay hidden unless Files are enabled, nmap defaults to TCP connect scans instead of unreliable raw-socket mode, and container smoke coverage now exercises workspace file reads/writes
  - consolidated external command policy, catalog metadata, autocomplete hints, runtime adaptations, seeded-history inputs, workflows, and smoke examples around `app/conf/commands.yaml`, while moving app-owned built-in autocomplete grammar into `app/builtin_autocomplete.yaml`; added ranked exact/prefix/boundary/substring/fuzzy autocomplete matching, explicit `value_type: domain` metadata so recent-domain capture and suggestions only apply to declared domain slots, configurable restricted IP/CIDR command-input validation for declared target slots, explicit `allow_grouping` metadata for safe POSIX-style short-flag bundles, ProjectDiscovery session-scoped runtime state via `XDG_CONFIG_HOME`, and a curated SecLists catalog plus `value_type: wordlist` metadata so installed wordlist autocomplete appears only in relevant command slots
  - expanded guided workflows with user inputs, sequential `Run all`, workspace-native chained recon examples that pass generated Files between tools, session-scoped user-created workflows, and the terminal-native `workflow list/show/run` command; added the `stats` built-in and refreshed seeded/demo/capture command sources from surfaced examples
  - added session command variables (`var set/list/unset`, `$NAME` / `${NAME}` expansion before validation, typed-history preservation, migration with session identity, and autocomplete for defined variables)
  - moved output review toward server-owned metadata: command findings, warnings, errors, summaries, scoped navigation, and transcript-native summaries now share persisted signal data
  - improved long-output and long-running reliability: faster high-volume rendering, incremental signal-count caching, no-signal output short-circuiting, chunked client-side multi-line replay for repeated workspace reads, absolute line-number preservation after trimming, partial-line stream heartbeats, a Redis-backed `/runs` broker foundation with replayable command events and active-run reload reattachment, bounded detached-run cleanup, active-run polling before stall failure, PID-checked reload reconnect after container/app restarts, client-aware active-run ownership for browsers sharing one session token with read-only attach and explicit takeover from Run Monitor, terminal `runs` / `jobs` active-run output with CPU percent / RSS-memory snapshots, a HUD-attached Run Monitor opened from desktop HUD pills or `Alt+R` / `Option+R` with circular CPU/RSS memory meters, initial CPU sample warmup/cache behavior, page-bounded history loading, changed-only History run comparison with suggested/manual run selection, day-grouped older-run pagination, inline changed-line highlighting, and one-click restore of both compared runs into fresh tabs, and an idle `0 active runs` state, plus a mobile Run Monitor peek that replaces Recents while the active tab is running
  - consolidated the theme system around canonical modal and chrome roles: `modal_bg`, `chrome_bg`, `chrome_header_bg`, `chrome_row_bg`, `chrome_row_hover_bg`, `chrome_control_bg`, `chrome_control_border`, `chrome_divider_color`, `chrome_shadow`, `inline_surface_bg`, plus action/selection text roles and shared derived tokens for repeated dropdown, scrollbar, search-highlight, signal-search, toast-shadow, and button primitive color transforms; retired unused pre-redesign tab theme keys from built-in defaults, shipped themes, generated examples, and `THEME.md`
  - added lightweight theme visual drift guardrails for shipped theme key parity, retired-token regression, `THEME.md` reference order/defaults, undefined CSS theme vars, hardcoded color literals, and populated preview cards across every shipped theme
  - replaced browser-native select popups in Options and History filters with the shared app-native dropdown primitive, while keeping the original `<select>` controls as the state and testability source
  - completed the frontend primitive pass for buttons, dropdowns, rows, chips, badges, form controls, drawers/sheets, HUD action cells, and mobile gesture handles; the button primitive allowlist now has no exceptions, and delete/clear entry points stay neutral until their confirmation modals show destructive actions
  - polished reload and confirmation flows: restored tabs pin to the prompt tail, inactive long tabs restore at the bottom when activated, Files view opens at the top, Redis HUD status clears stale online state, and terminal yes/no prompts ignore history navigation and autocomplete
  - tightened session identity and destructive flows: session-token revoke now requires transcript confirmation, session migration carries workspace files forward, and New Folder uses the app-native confirm/input modal instead of a browser prompt
  - polished built-in terminal output with shared ANSI helpers for status-like tables, aligned `file list -l` / `ls -l` output, `runs`, `stats`, shell-shaped summaries, and active-run metadata
  - updated visual/demo tooling: stable seeded demo sessions, light/dark capture passes, a capture review index, desktop Run Monitor telemetry scenes for demo/capture review, ordered docs drift checks, configuration-default documentation coverage, and current v1.6 docs/test appendix counts
  - added stable local test wrappers for pytest and Playwright so npm scripts use repo-owned environment/config defaults instead of repeating brittle inline command setup

- Why it changed
  - v1.6 turns the shell from a command launcher into a safer workspace-aware analysis surface where tools can read and write session files without exposing raw shell filesystem behavior
  - command definitions had spread across allowlists, autocomplete, seed data, smoke fixtures, and workflows; the branch reduces that drift by centering surfaced commands on the registry
  - real scanner output and reload scenarios exposed UX gaps around findings review, long-output performance, stale active-run metadata, and confirmation/autocomplete interaction
  - Docker-based smoke validation and visual review needed to catch more of the user-facing command surface before release
  - the branch review found a handful of worthwhile hardening items around workspace path quoting, symlink-safe reads, detached run cleanup, env/config drift warnings, and docs-order drift checks

## Risks

- Regressions, follow-up work, or tradeoffs
  - workspace storage is intentionally disabled by default; production deploys need the documented `./workspaces:/workspaces` mount and expected `995:995` ownership/mode model
  - workspace-aware command flags are explicit by design, so newly added file-producing scanner options must be declared before they can safely read/write session files
  - recent-domain suggestions are also explicit by design; newly added command flags or arguments need `value_type: domain` before the browser records or suggests recent domains for them
  - restricted command-input CIDRs only inspect literal values in metadata-known target slots; domain-to-IP DNS resolution is intentionally out of scope for the first pass
  - output-signal metadata now starts server-side; future classifier tuning should happen in backend tests first so restored history and live output stay consistent
  - container smoke coverage is broader and more realistic, but external tools can still have transient upstream failures; the retry/warmup logic reduces noise without making syntax/output assertions toothless
  - active-run PID checking is strongest on Linux where `/proc` start-time tracking is available; non-Linux behavior keeps the safer fallback paths used by the local test harness
  - detached-drain cleanup now has a hard ceiling; very long intentionally-disconnected commands should stay connected if continued live capture is required beyond the configured timeout plus grace
