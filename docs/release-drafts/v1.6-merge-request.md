# v1.6 Release Merge Request

## Summary

This merge request prepares darklab_shell v1.6 for release. It turns the app from a command runner with history into a safer workspace-aware analysis shell: users can manage session files, scanners can read and write declared workspace paths, workflows can be saved and run, active commands can be recovered across browsers, and Status Monitor becomes a first-class operational surface.

The largest pieces in this release are:

- app-managed per-session Files/workspaces
- workspace-aware scanner input/output flags
- the command registry consolidation around `commands.yaml`
- saveable workflows and the `workflow` CLI
- Redis-backed `/runs` streaming with replay/attach behavior
- Status Monitor replacing the older run-only monitor
- better autocomplete, wordlist, recent-domain, and restricted-input metadata
- richer output findings, run comparison, history pagination, and long-output performance
- theme/chrome primitive cleanup and stronger diagnostics

This document is intentionally release-shaped. For the full list of changes included in this release, see the v1.6 section in `CHANGELOG.md`.

## Added

- **Session Files and guarded workspaces**
  - Added app-managed per-session workspace storage with safe relative paths, quotas, symlink rejection, inactivity cleanup, and operator-controlled enablement.
  - Added Files browsing with folders, breadcrumbs, New Folder, New File, edit, view, download, delete, recursive folder deletion warnings, and manual refresh.
  - Added richer file previews for JSON, JSONL/NDJSON, CSV/TSV, XML, raw HTTP responses, and large line-oriented text.
  - Added terminal-native helpers: `file list`, `file show`, `file add`, `file edit`, `file download`, confirmed `file rm`, plus cwd-aware `cd`, `pwd`, `ls`, `cat`, `mkdir`, and confirmed `rm` when Files are enabled.

- **Workspace-aware scanner commands**
  - Added declared input/output flag rewriting into the guarded session workspace for common scanner workflows.
  - Added workspace-aware file flags for tools including `nmap`, `curl`, `wget`, `ffuf`, `gobuster`, `nuclei`, `pd-httpx`, `katana`, `naabu`, `subfinder`, `dnsx`, `masscan`, `sslscan`, `sslyze`, `testssl`, `nikto`, and `wpscan`.
  - Added ProjectDiscovery session-scoped runtime state and secondary output surfacing so response stores, screenshots, exports, logs, and resume/config state can appear in Files.
  - Added workspace-enabled smoke fixtures that create input files, run representative file read/write examples, verify generated output, and clean up through the Files API.

- **Command registry and autocomplete metadata**
  - Added `commands.yaml` as the shared model for visible external command policy, catalog metadata, autocomplete hints, examples, runtime adaptations, workspace file flags, smoke coverage, seeded history, and workflow command coverage.
  - Added `app/builtin_autocomplete.yaml` for app-owned built-in command grammar.
  - Added ranked exact, prefix, token-boundary, substring, and fuzzy matching across commands, subcommands, flags, arguments, workspace files, recent domains, and wordlists.
  - Added explicit value metadata for domains, wordlists, workspace files, grouped short flags, and restricted command-input validation.

- **Wordlists, recent domains, and restricted targets**
  - Added the `wordlist` built-in for curated installed SecLists discovery, search, copy-friendly paths, and `--all` full-tree mode.
  - Added `value_type: wordlist` and `wordlist_category` autocomplete slots so installed wordlists appear only where they make sense.
  - Added session-backed recent-domain capture and autocomplete through explicit `value_type: domain` slots.
  - Added `restricted_command_input_cidrs` support for blocking literal IP/CIDR values in declared target slots, including values inside URLs and app-managed target-list files.

- **User-created workflows**

- **Session variables**

  - Added `runs` / `jobs` active-run views with CPU, RSS memory, PID, elapsed time, verbose mode, and JSON output.

  - Added mobile access even when no local tab is actively running, so another browser's same-session run can be attached or killed.

## Changed

- **Files and command execution are now workspace-aware by design**
  - Workspace-enabled tabs track a current directory and resolve app-native file commands relative to it.
  - Short `ls` / `file list` output stays compact for humans, while pipe helpers receive one logical entry per line so `ls | grep txt` behaves like a normal terminal.
  - Workspace-only examples, flags, and hints stay hidden when Files are disabled.
  - Scanner commands run with the shared workspace group model so app-created and command-created files remain usable without world-readable permissions.

- **nmap now avoids unreliable raw-socket defaults**
  - nmap defaults toward TCP connect scan behavior where needed, keeping scans useful under the unprivileged `scanner` user inside Docker.

- **Command metadata now has one registry-shaped source**
  - The older split between allowlist data, autocomplete data, seeded examples, smoke examples, and workflow lists has been reduced.
  - Built-in grammar remains app-owned, while external command metadata lives in `commands.yaml` / `commands.local.yaml`.

- **Autocomplete now prioritizes useful matches**
  - Suggestions rank by match quality instead of strict prefix-only behavior.
  - Hints, placeholders, and value labels remain visible but no longer act as searchable candidates.
  - Recent domains and wordlists appear only in explicit metadata-known slots.

- **Active-run control uses the session token as the permission boundary**
  - The older takeover/control ceremony was removed.
  - Same-session browsers can attach to visible runs and intentionally kill them.
  - Other subscribed browsers receive clearer killed-by-another-browser messaging.

- **Status Monitor replaced the old Run Monitor model**
  - Desktop is now a modal instead of a drawer.
  - Mobile has a dedicated menu entry.
  - Idle Status Monitor states still show useful system/session information instead of an empty panel.

- **Long-output rendering and search now scale better**
  - The frontend avoids whole-transcript rescans for plain output.
  - Large output uses chunked replay and lazy highlighting paths.
  - File viewer and terminal search share large-output protections.
  - Line numbers preserve the true emitted-output count even after visual trimming.

- **History and recents no longer materialize everything**
  - Drawer and mobile recents requests are page-limited at the SQL layer.
  - Mobile recents preload the first page so the sheet opens at a stable height.
  - Exit-code `-15` is treated as externally terminated instead of failed for stats/graphs.

- **Theme/chrome styling now uses shared roles**
  - Modal-like surfaces share `modal_bg`.
  - Shell chrome surfaces share `chrome_*` roles.
  - Repeated color transforms now resolve through derived theme tokens.
  - Unused pre-redesign tab tokens and surface-specific one-offs were retired.

- **Browser-native selects were replaced where they mattered**
  - Options and History filters now render through the app-native dropdown primitive while preserving the original `<select>` as state owner and test fallback.

- **Destructive actions are quieter until confirmation**
  - History/Recents delete actions, Files Delete, and Options Clear use neutral entry buttons.
  - Confirmation modals keep the destructive styling at the actual point of confirmation.

- **Internal naming now matches the product model**
  - Legacy "fake command" and "run monitor" naming was moved to built-in command and Status Monitor terminology throughout current code, tests, docs, selectors, and generated classes.

- **Unit harnesses now use production source seams**
  - Pure helpers for autocomplete, app preferences, session identity, workspace paths, runner filters, history formatting, search summaries, and output prompt/signal transforms moved into small browser-visible core scripts.
  - Tests import those seams instead of extracting private functions from large UI controllers.

- **Demo recording standard changed to OBS**
  - The older screenshot-stitch and CDP experiment paths were retired.
  - Desktop and mobile wrappers now open a headed Chromium window, pause for OBS window selection, and then start/stop recording through OBS WebSocket.

## Fixed

- **Workspace permission and path safety**
  - Workspace-rewritten command arguments are shell-quoted before crossing the `sh -c` boundary.
  - Workspace reads/downloads use final-component no-follow opens where supported.
  - Startup warns when entrypoint workspace setup and app workspace config disagree.
  - Workspace repair no longer resets scanner-owned command folders before scanner repair.

- **Large files and large terminal buffers**
  - Repeated `cat <large_file>` and repeated workspace reads no longer cause whole-terminal signal rescans.
  - Large file preview/search avoids per-keystroke freezing through debouncing, worker-friendly scheduling, lazy highlighting, and large-search thresholds.
  - File viewer loading feedback appears before large reads, refreshes, and expensive Preview/Raw switches.
  - Auto-refresh is off by default and disabled for files larger than 1 MB.

- **Run lifecycle edge cases**
  - Reload reconnect prunes stale active commands after app/container restarts.
  - Quiet streams no longer fail a still-running backend command without checking active-run state.
  - Detached drains are bounded by timeout plus grace.
  - Partial-line tools can keep heartbeats flowing without waiting for a newline.
  - Closing the only tab resets the preserved tab's in-memory terminal state.

- **Status Monitor and active-run UI**
  - Initial CPU sampling no longer opens as a misleading permanent `n/a`.
  - Status Monitor active-run rows update in place instead of rebuilding the whole section every poll.
  - Non-active broker streams pause while Status Monitor is open to avoid browser connection starvation, then resume from the last event id.
  - Constellation, heatmap, treemap, pulse strip, popovers, sparse states, and mobile glow rendering were tuned for readability and performance.

- **History and comparison**
  - History drawer/mobile recents use bounded SQL queries.
  - Compare search now searches command text for compare candidates.
  - Compare modal focus and Escape behavior match the rest of the app.
  - Changed lines can highlight intra-line differences instead of showing near-identical timestamp/runtime lines as pure add/remove noise.

- **Autocomplete and command policy**
  - Placeholder/value-hint text no longer becomes a searchable autocomplete match.
  - FAQ allowed-command chips now list root commands instead of duplicated flag/subcommand variants.
  - FAQ command chips trigger autocomplete after loading into the prompt.
  - Safe grouped short-flag policy uses explicit `allow_grouping` metadata.

- **Mobile and modal behavior**
  - Mobile Files create/edit inputs use mobile-safe defaults.
  - Mobile output wraps correctly with timestamps and line numbers enabled.
  - Mobile sheets share a consistent grab-handle/backdrop close model.
  - Modal focus readiness is explicit, reducing Playwright race conditions.
  - File add/edit launched from CLI opens only the editor modal instead of also opening the Files browser behind it.

- **Diagnostics and docs drift**
  - Bandit false positives around dynamic SQLite identifiers were addressed through quote-escaped metadata identifiers where interpolation is unavoidable.
  - Shellcheck issues in Playwright helper scripts were fixed.
  - README project structure, configuration docs, test appendices, and test totals now have drift coverage.

## Rollout and Upgrade Notes

- Files/workspace storage remains disabled unless an operator enables it.
- Deployments enabling Files need the documented writable workspace mount and ownership/mode model.
- The production Compose override now includes `./workspaces:/workspaces`.
- `allowed_commands.txt` and `autocomplete.yaml` are replaced by the command registry model; local customization should move through `commands.local.yaml`.
- New workspace-aware file-producing flags must be declared before they can read/write session files.
- New domain/wordlist/restricted-input autocomplete slots should use explicit `value_type` metadata.
- Redis is recommended for production broker replay, but local/small deployments still have an in-process fallback.
- Active-run PID pruning is strongest on Linux, where `/proc` start-time tracking is available.

## Risks and Follow-Up

- Workspace-aware command coverage is intentionally explicit. Some useful scanner flags may still need metadata before they are available through Files.
- Restricted target validation blocks literal IP/CIDR values in declared slots, but it does not DNS-resolve domains.
- Output-signal classification now starts server-side; future classifier tuning should happen in backend tests first so live output, history, and permalinks stay consistent.
- Container smoke coverage is broader, but external tools can still fail transiently. The suite keeps retries/warmups without weakening syntax and output assertions.
- The Status Monitor visuals are intentionally richer than the rest of the app; future tuning should lean on capture review instead of brittle SVG/CSS unit assertions.
- Detached backend runs still have cleanup ceilings. Very long intentionally disconnected commands should stay attached if live capture must continue beyond timeout plus grace.
