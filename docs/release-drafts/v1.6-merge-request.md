# v1.6 Merge Request

## Summary

- What changed
  - added app-mediated per-session workspace storage, the Files modal, folder navigation, file view/edit/download/delete flows, recursive folder delete with count-aware warnings, and terminal-native `file` / `ls` / `cat` / confirmed `rm` helpers
  - made workspace-aware scanner examples first-class: declared file input/output flags are rewritten into the guarded session workspace, workspace-only examples stay hidden unless Files are enabled, and container smoke coverage now exercises workspace file reads/writes
  - consolidated external command policy, catalog metadata, autocomplete hints, seeded-history inputs, workflows, and smoke examples around `app/conf/commands.yaml`
  - expanded guided workflows with user inputs and sequential `Run all`, added the `stats` built-in, and refreshed seeded/demo/capture command sources from surfaced examples
  - added session command variables (`var set/list/unset`, `$NAME` / `${NAME}` expansion before validation, typed-history preservation, migration with session identity, and autocomplete for defined variables)
  - moved output review toward server-owned metadata: command findings, warnings, errors, summaries, scoped navigation, and transcript-native summaries now share persisted signal data
  - improved long-output and long-running reliability: faster high-volume rendering, partial-line stream heartbeats, bounded detached-run cleanup, active-run polling before stall failure, PID-checked reload reconnect after container/app restarts, terminal `runs` / `jobs` active-run output with CPU percent / RSS-memory snapshots, and a HUD-attached Run Monitor opened from desktop HUD pills or `Alt+R` / `Option+R` with circular CPU/RSS memory meters and an idle `0 active runs` state
  - polished reload and confirmation flows: restored tabs pin to the prompt tail, inactive long tabs restore at the bottom when activated, Files view opens at the top, Redis HUD status clears stale online state, and terminal yes/no prompts ignore history navigation and autocomplete
  - updated visual/demo tooling: stable seeded demo sessions, light/dark capture passes, a capture review index, ordered docs drift checks, and current v1.6 docs/test appendix counts

- Why it changed
  - v1.6 turns the shell from a command launcher into a safer workspace-aware analysis surface where tools can read and write session files without exposing raw shell filesystem behavior
  - command definitions had spread across allowlists, autocomplete, seed data, smoke fixtures, and workflows; the branch reduces that drift by centering surfaced commands on the registry
  - real scanner output and reload scenarios exposed UX gaps around findings review, long-output performance, stale active-run metadata, and confirmation/autocomplete interaction
  - Docker-based smoke validation and visual review needed to catch more of the user-facing command surface before release
  - the branch review found a handful of worthwhile hardening items around workspace path quoting, symlink-safe reads, detached run cleanup, env/config drift warnings, and docs-order drift checks

## Risks

- Regressions, follow-up work, or tradeoffs
  - workspace storage is intentionally disabled by default; production deploys need the documented `./workspaces:/workspaces` mount and expected `995:995` ownership/mode model
  - workspace-aware command flags are explicit by design, so newly added file-producing scanner options must be declared before they can safely read/write session files
  - output-signal metadata now starts server-side; future classifier tuning should happen in backend tests first so restored history and live output stay consistent
  - container smoke coverage is broader and more realistic, but external tools can still have transient upstream failures; the retry/warmup logic reduces noise without making syntax/output assertions toothless
  - active-run PID checking is strongest on Linux where `/proc` start-time tracking is available; non-Linux behavior keeps the safer fallback paths used by the local test harness
  - detached-drain cleanup now has a hard ceiling; very long intentionally-disconnected commands should stay connected if continued live capture is required beyond the configured timeout plus grace
