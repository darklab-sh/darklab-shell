# v2.0 Release Merge Request

## Summary

- Adds first-pass interactive PTY support for screen-oriented tools, rendered in an xterm.js modal and brokered through Redis-capable PTY routes.
- Persists PTY completion output as normal saved run history/search/finding content while keeping live redraw output out of the parent transcript.
- Adds database preview-size protection, startup query indexes, and two terminal/history polish fixes found during v2.0 work.

## Added

- Interactive PTY lifecycle for configured registry commands, including `mtr --interactive`, `ffuf --interactive`, and `masscan --interactive`.
- Vendored xterm.js terminal rendering with fit-addon support for the live PTY modal.
- xterm.js and its fit addon preload during app startup when interactive PTY mode is enabled, reducing the visible delay before the first PTY modal is usable.
- Failed xterm.js vendor asset loads can now be retried cleanly, and live PTY terminals refresh their palette when the app theme changes.
- Mobile terminal mode denies interactive PTY shell startup with a desktop-only support message before opening xterm or creating a backend run.
- Server-side PTY terminal capture for persisted static output after interactive runs finish.
- PTY startup now fails with a clear dependency message if `pyte` is unavailable while interactive mode is enabled, rather than silently falling back to lossy capture.
- Tab-scoped PTY modal ownership so the live terminal covers only its owning tab panel, not the whole application.
- PTY modal close now uses the same running-tab confirmation flow as normal tabs, including Cancel, Keep running, and Kill options.
- Multiple concurrent PTY runs per browser session, with each live terminal scoped to the tab that started it.
- Consistent sizing and exit cleanup for concurrent PTY modals, including runs that finish after another PTY tab has become active.
- PTY reattach from reload recovery and Status Monitor Attach using an ANSI terminal snapshot plus live stream resume from the snapshot event id.
- Server-side PTY snapshots preserve terminal colors, text attributes, cursor position, and the latest visible screen when pyte capture is available, with a plain-text fallback for degraded cases.
- Distributed Redis PTY snapshots so any worker can serve reload recovery and Status Monitor Attach without sticky routing; failed attaches still avoid consuming new tabs if a snapshot is not available yet.
- PTY keyboard input now forwards native `Ctrl+C` interrupts to the interactive process while the modal kill and close buttons keep using the standard kill confirmation modal.
- PTY input and resize routes now use the shared rate limiter, and browser-side input truncation matches the backend UTF-8 byte limit before posting.
- Interactive PTY commands inherit only a vetted subset of the process environment, preserving practical terminal variables without leaking broad host/process state.

## Changed

- Run previews stored in SQLite are capped by `output_preview_max_mb` in addition to existing line limits.
- Recent-command, snapshot, and workflow startup queries use targeted session-aware SQLite indexes.
- Recent domain capture now preserves complete IPv4 addresses from domain-style argument slots while rejecting partial numeric hosts.

## Fixed

- Restoring a history run now reuses a tab only when the run identity matches, not merely when the command text matches.
- macOS double-space period substitution is normalized out of the command composer before it updates autocomplete or draft state.

## Validation

- Focused validation is still in progress for the full v2.0 branch.
