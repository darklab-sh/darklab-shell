# v2.0 Release Merge Request

## Summary

- Adds the project workspace backend foundation: relationship tables, entity/link-source validation, supporting indexes, initial project CRUD/link routes, active project context, automatic saved-run linking, and session migration support for future project, artifact, target, finding, label, annotation, and package flows.
- Adds first-pass interactive PTY support for screen-oriented tools as a cohesive, guarded feature: configured commands run in a tab-scoped xterm.js modal, route through dedicated Redis-capable PTY endpoints, and persist completed output as normal saved history/search/finding content.
- Ships PTY with first-pass safeguards for registry-owned command settings, bounded concurrency, input policy, reattach snapshots, transcript shaping, route rate limits, stale-state cleanup, and multi-worker operation.
- Adds database preview-size protection, startup query indexes, and two terminal/history polish fixes found during v2.0 work.

## Added

- Project workspace relationship tables for projects, project links, run file artifacts, project targets, persisted findings, entity labels, and annotations.
- App-owned project entity/link-source constants plus bootstrap/index tests so project routes share one relationship vocabulary.
- Session-scoped `/projects` routes for project CRUD, active project context, archive filtering, supported entity link/unlink, project link listing, and source-record ownership checks.
- Automatic project links for completed server-owned runs, browser-owned built-ins, and snapshots created from project-associated tabs.
- Project-aware `/history?project_id=...` filters for linked runs and snapshots.
- Session-token migration support for projects, run file artifacts, findings, entity labels, and annotations.
- Interactive PTY lifecycle for configured registry commands, including `mtr --interactive`, `ffuf --interactive`, and `masscan --interactive`, with registry-owned trigger flags, terminal dimensions, input policy, input-safety profile, runtime limit, and transcript mode.
- Dedicated `/pty/runs` start/stream/input/resize routes with Redis-brokered output/input/resize events for multi-worker deployments and a single-worker local fallback.
- Vendored xterm.js terminal rendering with fit-addon support, startup preloading, retryable asset loading, live theme refresh, reusable tab-scoped overlays, hidden-tab resize suppression, and desktop-only mobile startup guardrails.
- Tab-scoped PTY modal ownership with normal running-tab close/kill confirmation behavior, bounded per-session concurrency, consistent concurrent modal sizing, and clean exit handling for each PTY tab.
- Server-side pyte terminal capture for completed static output plus bounded ANSI snapshots for reload recovery and Status Monitor Attach. Redis-backed snapshots are distributed, rate-capped, and have specific failure statuses for missing, closed, stale, or not-yet-available runs.
- Registry-owned completed transcript shaping so `mtr` saves the final frame while `ffuf` and `masscan` save scrolled findings without common transient redraw noise.
- PTY safety guardrails covering explicit input-safety profiles, native `Ctrl+C` forwarding, rate-limited control routes, byte-aware input truncation, workspace-CWD-aware validation, vetted command environment inheritance, clear terminal-capture dependency errors, kill notices through the PTY stream, owner-liveness refresh, startup cleanup, and stale Redis metadata pruning.

## Changed

- Run previews stored in SQLite are capped by `output_preview_max_mb` in addition to existing line limits.
- Recent-command, snapshot, and workflow startup queries use targeted session-aware SQLite indexes.
- Recent domain capture now preserves complete IPv4 addresses from domain-style argument slots while rejecting partial numeric hosts.

## Fixed

- Restoring a history run now reuses a tab only when the run identity matches, not merely when the command text matches.
- macOS double-space period substitution is normalized out of the command composer before it updates autocomplete or draft state.

## Validation

- Focused validation is still in progress for the full v2.0 branch.
