# darklab_shell v1.7 Release Notes

v1.7 rounds out the Files workflow. Session files are now easier to organize from the modal and easier to work with from the terminal.

For the full list of changes included in this release, see the v1.7 section in [CHANGELOG.md](../../CHANGELOG.md).

## Highlights

### Move and Rename Session Files

Files can now be moved or renamed without leaving darklab_shell.

- Use the Files modal row action to move a file or folder.
- Drag a file onto a folder row to move it there.
- Run `file move <source> <destination>` from the terminal.
- Use `mv <source> <destination>` when you want the shorter shell-style form.

Moves stay inside your current session workspace. The app rejects unsafe paths, folder self-moves, symlinks, and accidental overwrites.

### Simple `*` Patterns for Workspace Commands

Common workspace commands now understand simple `*` patterns:

```bash
file ls darklab-*
mv darklab-* darklab/
file delete scan-*
```

The pattern is intentionally small and predictable: `*` matches inside one path segment and does not cross `/`. If a move matches more than one item, the destination must already be a folder. Deletes still ask for confirmation, and folders still require `-r` or `-rf`.

### Better File Autocomplete

Autocomplete now helps with the new move flow. `file move` and `mv` suggest real session files and folders for the source, then suggest destination folders once a source has been chosen.

The `file list` / `file ls` suggestions were also cleaned up so the menu shows one useful row for each command instead of duplicate entries.

## Testing

The release adds backend, Vitest, and Playwright coverage for the new Files behavior, including move validation, drag/drop, terminal moves, glob list/move/delete flows, and end-to-end modal-to-terminal use.
