# Allowed Commands
# -----------------
# One allowed command prefix per line.
# Lines starting with # are comments and are ignored, as are blank lines.
# Lines starting with ## define a category group shown in the FAQ command list.
# Matching is prefix-based: "ping" allows "ping google.com", "ping -c 4 1.1.1.1", etc.
# Be as specific or broad as you like — "nmap -sT" only allows TCP-connect scans,
# while "nmap" would allow any nmap invocation.
#
# Lines starting with ! are deny prefixes and take priority over allow prefixes.
# Use them to block specific flags on an otherwise-allowed command. For example:
#   nmap
#   !nmap -sU
#   !nmap --script
# This allows all nmap commands except those containing -sU or --script as a flag.
# Unlike allow entries, deny flags are matched anywhere in the command (not just
# immediately after the tool name), so "nmap -sT -sU 10.0.0.1" is also blocked.
# Exception: a denied output flag is allowed when its argument is /dev/null, so
# patterns like "curl -o /dev/null -w "%{http_code}" <url>" still work.
# Deny prefixes are not shown to users in the FAQ command list.
#
# Note on mtr: mtr's interactive full-screen mode requires a real TTY which is not
# available here. The app automatically rewrites "mtr <target>" to
# "mtr --report-wide <target>" so it runs in non-interactive report mode instead.
# If you include "mtr --report" or "mtr -r" as a prefix here, users must pass
# report flags themselves — the auto-rewrite only fires when they are absent.
