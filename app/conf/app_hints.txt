# Rotating app hints shown in the welcome footer.
# These are application-focused tips only, not command examples.

Use the history panel to reopen saved runs.
Click a permalink to share command output.
Autocomplete only suggests allowed diagnostics commands.
The run button executes the current command in the prompt.
Long output is trimmed so the terminal stays responsive.
Arrow Up/Down cycles command history when autocomplete is not active.
The welcome banner uses sampled examples from the allowlist.
Use the search bar to scan output with case-sensitive or regex matching.
Built-in pipe support lets commands use | grep, | head, | tail, and | wc -l.
Double-click a tab label to rename it without losing later command output.
The timestamps toggle cycles between off, elapsed, and clock time.
The line number toggle makes long output easier to reference and share.
Save HTML preserves ANSI colors; save txt exports a plain text snapshot.
Use share snapshot when you want the current tab view, not just one saved run.
Run permalink in the history panel copies the canonical link for a single saved command.
The kill button sends SIGTERM to the whole process group for that tab.
If you scroll away from a live run, use jump to live to snap back to the tail.
The history chips row lets you reload recent commands with one click.
Star a history entry to pin it to the top of the history drawer and recent chips.
Use Ctrl+R for reverse-history search when you remember part of an earlier command.
Tab expands shared autocomplete prefixes before it starts cycling suggestions.
Shift+Tab walks autocomplete suggestions in reverse.
After `nmap ` or `curl -`, autocomplete can switch to command-specific flags and argument hints.
Context-aware autocomplete shows positional hints too, so `nmap ` can suggest both flags and `<target>`.
Reload keeps idle tabs, restored output, and draft text in the current browser session.
If a run is still active after reload, the tab reconnects and swaps in the saved result when it finishes.
History filters can narrow runs by search text, command root, exit code, date, and starred state.
Active history filters show up as removable chips so you can clear one without resetting the rest.
Older history entries show their date alongside the time so searches are easier to scan.
Copy, save txt, and save html all work from the active tab without leaving the shell.
Theme changes apply to the shell, history overlays, and shared snapshot pages.
Press any key to end the welcome animation early and jump to the prompt.
Run the command 'help' to view the shell's built-in commands.
Run the command 'ls' to view all available commands.
