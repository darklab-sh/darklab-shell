FAQ
What is this?
shell.darklab.sh is a lightweight web interface for running network diagnostic and vulnerability scanning commands against remote endpoints, with output streamed in real time. It's designed for testing and troubleshooting remote hosts — things like DNS lookups, port scans, traceroutes, HTTP checks, and web app vulnerability scans — without needing SSH access to a server. For more detailed information, see the README on GitLab.

Why does mtr look different here?
mtr requires a real terminal (TTY) for its live interactive display, which isn't available in a web shell. It's automatically run in --report-wide mode instead, which runs 10 probe cycles and prints a summary table. You can control the cycle count with -c, e.g. mtr -c 20 google.com
Can I request a new tool?
Yes! If there's a tool you'd like added, send a request to admin [at] darklab [dot] sh and we'll consider it for a future update.
How do tabs and permalinks work?
Each command runs in the currently active tab. You can open additional tabs with the + button to run commands side by side and keep results from different sessions visible at the same time. Each tab tracks its own status independently. Double-click a tab label to rename it.

The permalink button on each tab captures the current tab output, and when a full saved artifact exists it shares that full output as a styled HTML page. The link opens a view with ANSI color rendering and options to copy to clipboard, save as .html, save as .txt, or view raw JSON. Permalinks survive container restarts.

The ⧖ history panel shows your last 50 runs. You can load any past result into a new tab, copy a single-run permalink from there, or ★ star a command to pin it to the top of the list.
How do I stop a running command?
Click the ■ Kill button that appears while a command is running. This sends SIGTERM to the entire process group on the server, stopping it immediately.
How do I access search, history and theme on mobile?
On small screens the header buttons are replaced by a ☰ menu in the top-right corner. Tap it to access search, run history, line numbers, timestamps, theme, and this FAQ.
How do I save or share my results?
There are several options from the action bar below each tab's output:

permalink — saves the current tab output and generates a shareable URL. If a full saved artifact exists, the link uses that full output; otherwise it uses the current tab snapshot. The page lets the recipient copy, download, or view the raw data.
copy — copies the full plain-text output to your clipboard.
save .html — downloads a themed HTML file with ANSI colors preserved. It uses app-hosted vendor fonts when opened alongside this shell and falls back to browser monospace fonts offline.
save .txt — downloads a plain-text version of the output.

Single-run permalinks are also available from the ⧖ history panel.
How do I rename a tab?
Double-click the tab label to edit it inline. Press Enter or click anywhere outside to confirm, or Escape to cancel. Once renamed, running a command won't overwrite the label — the tab keeps your chosen name.
What do the timestamp options do?
The timestamps button in the terminal bar cycles through three modes:

off — no timestamps shown (default).
elapsed — shows how many seconds after the command started each line appeared (e.g. +4.2s). Useful for understanding how long different stages of a scan take.
clock — shows the wall-clock time each line was received (e.g. 14:32:01). Useful for correlating output with events elsewhere.
What do the line number options do?
The line numbers button in the terminal bar toggles numbered output lines on and off.

off — no line numbers are shown (default).
on — every output line is prefixed with a sequence number so you can reference specific rows while reading long scans or copied output.
Are there keyboard shortcuts?
Yes. Current shell-style shortcuts include:

Ctrl+C — open kill confirmation while a command is running, or drop to a fresh prompt line when idle.
Enter on a blank prompt — create a new empty prompt line.
Option+T / Option+W (Alt+T / Alt+W) — open or close the current tab.
Option+←/→ and Option+1...9 (Alt+←/→, Alt+1...9) — switch tabs.
Option+P (Alt+P) — create a permalink for the active tab.
Option+Shift+C (Alt+Shift+C) — copy the active tab output.
Ctrl+L — clear the active tab.
Kill dialog: Enter confirms and Escape cancels.
Ctrl+W, Ctrl+U, Ctrl+K, Option+B, and Option+F (Alt+B, Alt+F) provide readline-style prompt editing.
Welcome screen: printable typing, Enter, and Escape all settle the welcome animation immediately.
Autocomplete: ↑↓ navigate, Tab accepts, Enter accepts or runs, and Escape dismisses.

On macOS, the app-safe tab/action shortcuts use the Option key. The Ctrl+... bindings are intentional shell-style controls, not replacements for browser Command shortcuts.

You can also run shortcuts in the terminal for the current shortcut reference.
Are my commands visible to other users?
No. Each browser session is assigned an anonymous ID stored in your browser's local storage. Your run history, starred commands, and saved snapshots are only visible to sessions sharing that ID — in practice, just your own browser tabs. Commands are not broadcast or shared between users.

What wordlists are available?
The full SecLists collection is installed at /usr/share/wordlists/seclists/. Commonly used lists:
  • Discovery/Web-Content/common.txt — fast directory scan
  • Discovery/Web-Content/big.txt — broader directory scan
  • Discovery/Web-Content/DirBuster-2007_directory-list-2.3-big.txt — thorough directory scan
  • Discovery/Web-Content/raft-large-files.txt — common file names
  • Discovery/DNS/subdomains-top1million-5000.txt — fast subdomain brute-force
  • Discovery/DNS/subdomains-top1million-20000.txt — broader subdomain brute-force
  • Discovery/DNS/subdomains-top1million-110000.txt — thorough subdomain brute-force
  • Passwords/Common-Credentials/10-million-password-list-top-10000.txt — common passwords
  • Usernames/Names/names.txt — common usernames
  • Fuzzing/ — XSS, SQLi, path traversal, and other fuzzing payloads
